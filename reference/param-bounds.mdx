Every parameter returned in `ModelOut.params` is validated and clamped against the bounds defined in `ModeContract.modeBounds`. Params outside their bounds are clamped silently and logged as interventions. Non-finite values and unknown param names are hard violations that trigger safe defaults for the entire output.

## Default bounds

All parameters default to `[0.0, 1.0]` unless a mode-specific override is listed below. Overrides only apply to the mode they are defined for â€” the same param name in a different mode keeps the default `[0.0, 1.0]` bounds.

<Note>
Out-of-range params are clamped silently and logged with a `param_clamped_low:` or `param_clamped_high:` violation string. These are soft violations â€” clamped output is still used. Non-finite and unknown params are hard violations that replace the entire output with safe defaults.
</Note>

## Clamping and violation behaviour

| Condition | Violation type | Effect |
|---|---|---|
| Param value below lower bound | `param_clamped_low:<key>:<raw>-><clamped>` | Soft â€” clamped value used, logged |
| Param value above upper bound | `param_clamped_high:<key>:<raw>-><clamped>` | Soft â€” clamped value used, logged |
| Non-finite param value (`NaN`, `Inf`) | `param_not_finite:<key>` | **Hard** â€” safe defaults for entire output |
| Unknown param name | `param_not_allowed:<key>` | **Hard** â€” safe defaults for entire output |
| Density cap triggered | `density_cap` | Soft â€” informational |
| Voice cap triggered | `voice_cap` | Soft â€” informational |

<Warning>
Hard violations (`param_not_finite`, `param_not_allowed`, `protocol_version_mismatch`, `mode_mismatch`, `missing_pick`) cause `ModeContract.enforceIncoming` to discard the entire `ModelOut` and substitute safe defaults. Soft violations log but pass through.
</Warning>

## Default value for missing params

When a param is missing from `ModelOut.params`, the harness fills it with the midpoint of its bounds: `(min + max) / 2`. Per-mode `safeModeParams` override this midpoint with tuned defaults when safe defaults are applied due to a hard violation.
