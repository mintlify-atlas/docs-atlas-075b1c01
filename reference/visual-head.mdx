`VisualOut` is the visual control signal embedded in every `ModelOut`. It is required in protocol v2 â€” its absence is a decode error. `VideoOutputController` maps `VisualOut` fields to scene parameters, and `VideoStageView` renders the result at 24 Hz using SwiftUI Canvas.

## VisualOut fields

<ParamField path="scene_id" type="string" required>
  One of 6 valid scene identifiers. Unknown values fall back to the mode default. See [Scene IDs](#scene-ids) below.
</ParamField>

<ParamField path="density" type="number" required>
  Visual density `[0, 1]`. Higher values produce denser, more active scenes. Clamped to `[0.0, 1.0]`.
</ParamField>

<ParamField path="cohesion" type="number" required>
  Visual cohesion `[0, 1]`. Controls how unified and structured the scene elements appear. Clamped to `[0.0, 1.0]`.
</ParamField>

<ParamField path="disruption" type="number" required>
  Visual disruption `[0, 1]`. Controls chaos, fragmentation, and instability in the scene. Clamped to `[0.0, 1.0]`.
</ParamField>

<ParamField path="token_salience" type="number" required>
  Token salience `[0, 1]`. Drives the prominence of text/token elements in the scene. Clamped to `[0.0, 1.0]`.
</ParamField>

<ParamField path="wordmark_integrity" type="number" required>
  Wordmark integrity `[0, 1]`. Controls the legibility and presence of the wordmark. `1.0` = fully intact. Clamped to `[0.0, 1.0]`.
</ParamField>

<ParamField path="decay_ms" type="number" required>
  Visual decay time in milliseconds `[120, 2400]`. Controls how quickly scene elements fade. Clamped to `[120, 2400]` as an integer.
</ParamField>

<ParamField path="flash_bias" type="number" required>
  Flash bias `[0, 1]`. Higher values increase the probability and intensity of flash events. Clamped to `[0.0, 1.0]`.
</ParamField>

<ParamField path="anchor_weights" type="number[]" required>
  3-element array of normalised weights `[w0, w1, w2]`. Controls relative weighting between three spatial anchor regions. Must sum to approximately `1.0`. See [Sanitization](#sanitization) for fallback behaviour.
</ParamField>

<ParamField path="thought" type="string" required>
  Current model thought from the 20-value vocab. Unknown values fall back to `"idle"`. See [Thought vocab](#thought-vocab) below.
</ParamField>

<ParamField path="thought_log" type="string[]">
  Recent history of thought transitions. Optional; defaults to `[]` on decode if absent.
</ParamField>

## Scene IDs

Six scene identifiers are valid. Any unknown value causes `ModeContract.sanitize` to substitute the mode-default scene.

| Scene ID | Character |
|---|---|
| `grid_lock` | Structured, locked grid. High cohesion, low disruption. |
| `relay_mesh` | Interconnected mesh with relay-like transitions. Balanced. |
| `fault_lattice` | Fragmented, faulted lattice. High disruption. |
| `memory_drift` | Drifting, diffuse memory texture. Mid density, low cohesion. |
| `alarm_splay` | Splayed, high-energy alarm state. High density and disruption. |
| `quiet_watch` | Quiet, minimal observation state. Low density, high integrity. |

## Sanitization

`ModeContract.sanitize(visual:mode:)` is called on every incoming `VisualOut` before it reaches the stage. It applies the following rules in order:

| Field | Invalid condition | Sanitization |
|---|---|---|
| `scene_id` | Not in `visualSceneIds` | Replaced with `VisualOut.defaultForMode(mode).sceneId` |
| `density` | Non-finite or out of `[0, 1]` | Non-finite â†’ midpoint `0.5`; out-of-range â†’ clamped |
| `cohesion` | Non-finite or out of `[0, 1]` | Same as above |
| `disruption` | Non-finite or out of `[0, 1]` | Same as above |
| `token_salience` | Non-finite or out of `[0, 1]` | Same as above |
| `wordmark_integrity` | Non-finite or out of `[0, 1]` | Same as above |
| `flash_bias` | Non-finite or out of `[0, 1]` | Same as above |
| `decay_ms` | Outside `[120, 2400]` | Clamped to `[120, 2400]` |
| `anchor_weights` | Count â‰  3 | Replaced with fallback `[0.34, 0.33, 0.33]` |
| `anchor_weights` | Any element non-finite or negative | Replaced with fallback `[0.34, 0.33, 0.33]` |
| `anchor_weights` | Sum â‰¤ 0.0001 | Replaced with fallback `[0.34, 0.33, 0.33]` |
| `anchor_weights` | Valid but not summing to 1.0 | Normalised: each weight divided by total sum |
| `thought` | Not in `visualThoughtVocab` | Replaced with `"idle"` |

Each sanitization step appends a violation string (e.g. `visual_scene_fallback:unknown_id`, `visual_clamped:density`, `visual_not_finite:cohesion`, `visual_anchor_weight_count:2`, `visual_invalid_thought:hallucinating`). Visual violations are soft â€” sanitized output is used rather than falling back to safe defaults.

```swift
// Anchor weight normalization in ModeContract.sanitize
let weightTotal = weights.reduce(0, +)
let anchorWeights = weightTotal > 0.0001
    ? weights.map { max(0, $0) / weightTotal }
    : fallback.anchorWeights
```

## Rendering pipeline

`VisualOut` flows from the contract layer to the screen via two components:

**`VideoOutputController`** â€” receives `VisualOut` from `TubMLClient` after enforcement and maps fields to scene-specific rendering parameters. It maintains a `@Published` state that drives the visual stage.

**`VideoStageView`** â€” a SwiftUI `Canvas`-based view that renders at 24 Hz. It reads from `VideoOutputController` and paints the active scene using the mapped parameters. The 24 Hz render loop is independent of the 10 Hz model loop; the most recent `VisualOut` is held and re-rendered each frame until a new one arrives.

<Info>
The visual stage renders at 24 Hz regardless of the 10 Hz model frame rate. `VideoStageView` interpolates between successive `VisualOut` values to produce smooth transitions.
</Info>
