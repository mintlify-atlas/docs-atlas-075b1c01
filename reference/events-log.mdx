---
title: "Events Log"
description: "Format of events_<session_id>.jsonl — session lifecycle events including the bundle header and human label changes."
---

The events log records discrete events that occur during a session, separate from the 10 Hz frame stream. It is a JSONL file: one JSON object per line, written in chronological order.

Unlike the frame log, events are sparse — they are written only when something significant changes. The two event types currently defined are the bundle header (emitted once at session open) and label change events (emitted whenever the human label changes via the UI or keyboard).

---

## Event types

### `bundle_header`

Emitted **once**, immediately when the session is created, before any frames are written. Records the full run bundle metadata so that any downstream consumer of the events file has all version and contract information without needing to look up the bundle file separately.

#### Example

```json
{
  "ts_ms": 1711334401000,
  "event": "bundle_header",
  "bundle_id": "bundle_2026-03-24_a1b2c3-policyabc",
  "session_id": "2026-03-24T10-05-00Z_a1b2c3",
  "bundle": {
    "bundle_id": "bundle_2026-03-24_a1b2c3-policyabc",
    "created_at": "2026-03-24T10:05:00Z",
    "policy_version": "sha256:abcdef01...",
    "bank_manifest_version": "sha256:12345678...",
    "contract_version": "control_surface_v2",
    "contract_fingerprint": "3394bcdfa3c207eb33da3679ee7d15d9f017bba1ab2da6567a930947f9fb586f",
    "harness_repo_sha": "a1b2c3d",
    "model_repo_sha": "e4f5a6b"
  }
}
```

#### Fields

<ResponseField name="ts_ms" type="integer">
  Event timestamp in milliseconds since Unix epoch.
</ResponseField>

<ResponseField name="event" type='"bundle_header"'>
  Event type discriminator. Always `"bundle_header"` for this event.
</ResponseField>

<ResponseField name="bundle_id" type="string">
  The active run bundle ID. Repeated at the top level for quick lookup without traversing the nested `bundle` object.
</ResponseField>

<ResponseField name="session_id" type="string">
  The session ID this event belongs to.
</ResponseField>

<ResponseField name="bundle" type="object">
  Full run bundle metadata payload.

  <Expandable title="bundle fields">
    <ResponseField name="bundle_id" type="string">
      Run bundle identifier. Encodes date, harness git short SHA, and a hash prefix of the policy config: `bundle_<YYYY-MM-DD>_<sha7>-<policy8>`.
    </ResponseField>
    <ResponseField name="created_at" type="string (ISO 8601)">
      Timestamp at which the bundle was created.
    </ResponseField>
    <ResponseField name="policy_version" type="string">
      SHA-256 hex digest of the policy config YAML file in use, or `"unknown_policy_version"` if the file was not found. Set by `TUB_POLICY_VERSION` environment variable if present.
    </ResponseField>
    <ResponseField name="bank_manifest_version" type="string">
      SHA-256 hex digest of the concatenated manifest files (`banks.json`, `instruments.json`, `chords.json`, `motifs.json`, `spatial_patterns.json`). Changes whenever any manifest is updated.
    </ResponseField>
    <ResponseField name="contract_version" type="string">
      Protocol contract version string (e.g. `"control_surface_v2"`). Must match between harness and model server.
    </ResponseField>
    <ResponseField name="contract_fingerprint" type="string">
      SHA-256 hex fingerprint of the mode contract tables (canonical params, bounds, aliases, required picks, safe defaults, visual schema). Must match `ModeContract.lockedContractFingerprint`. A mismatch between harness and model is a release blocker. See [Testing](/reference/testing).
    </ResponseField>
    <ResponseField name="harness_repo_sha" type="string | null">
      7-character git short SHA of the harness repository at build time. `null` when the repository is not a git repo or the SHA could not be resolved.
    </ResponseField>
    <ResponseField name="model_repo_sha" type="string | null">
      7-character git short SHA of the model repository, read from the `TUB_ML_GIT_SHA` environment variable at launch time. `null` if the variable is not set.
    </ResponseField>
  </Expandable>
</ResponseField>

### `label_change`

Emitted whenever the human operator changes the sticky quality label for the current session. This includes changes via the UI buttons, keyboard shortcuts (1 = good, 2 = too_much, 3 = too_flat, 0 = clear), and programmatic label updates.

The label is also embedded in every frame log line as `label`. The events log provides a sparse timeline of label transitions, which is useful for segmenting sessions by quality annotation without parsing the full frame log.

Label change events can also be written **after a session closes** via a detached append (e.g. when the operator labels a past session from the session browser). In that case the session's events file is opened, seeked to the end, and the event is appended.

#### Example

```json
{
  "ts_ms": 1711334415820,
  "event": "label_change",
  "from": null,
  "to": "good",
  "bundle_id": "bundle_2026-03-24_a1b2c3-policyabc",
  "session_id": "2026-03-24T10-05-00Z_a1b2c3"
}
```

```json
{
  "ts_ms": 1711334432100,
  "event": "label_change",
  "from": "good",
  "to": "too_much",
  "bundle_id": "bundle_2026-03-24_a1b2c3-policyabc",
  "session_id": "2026-03-24T10-05-00Z_a1b2c3"
}
```

```json
{
  "ts_ms": 1711334450300,
  "event": "label_change",
  "from": "too_much",
  "to": null,
  "bundle_id": "bundle_2026-03-24_a1b2c3-policyabc",
  "session_id": "2026-03-24T10-05-00Z_a1b2c3"
}
```

#### Fields

<ResponseField name="ts_ms" type="integer">
  Event timestamp in milliseconds since Unix epoch. Recorded at the moment the label change was applied.
</ResponseField>

<ResponseField name="event" type='"label_change"'>
  Event type discriminator. Always `"label_change"` for this event.
</ResponseField>

<ResponseField name="from" type='"good" | "too_much" | "too_flat" | null'>
  The label value before this change. `null` when transitioning from the unlabeled state.
</ResponseField>

<ResponseField name="to" type='"good" | "too_much" | "too_flat" | null'>
  The label value after this change. `null` when the operator cleared the label (keyboard shortcut `0` or `Clear Label`).
</ResponseField>

<ResponseField name="bundle_id" type="string">
  The active run bundle ID at the time of the label change.
</ResponseField>

<ResponseField name="session_id" type="string">
  The session ID this label change belongs to.
</ResponseField>
